Section 1 — What Is It

01

What is Figma MCP?

Model Context Protocol is an open standard for exposing tools to AI agents. Figma implements it so agents can call your design files like any other API.

↓ to dive in
What Is It · Concept

MCP in one diagram

── Without MCP ──────────────────────────────────
You → copy hex from inspector → paste into prompt → AI guesses layout
Result: wrong padding, wrong font weight, three correction rounds

── With Figma MCP ───────────────────────────────
Agent → get_design_context(fileKey, nodeId) → receives full token tree
Agent → generates component with exact values from your file
Result: pixel-perfect code, first attempt
Key point: The agent calls Figma's API directly through MCP tools. No screenshots, no manual context transfer, no prompt stuffing.
What Is It · Variants

Remote vs. Desktop MCP

🌐 Remote (recommended)

  • Hosted by Figma — no local process
  • Broadest feature set
  • Works without the desktop app
  • Config-only setup

🖥 Desktop

  • Runs via Figma desktop app
  • Enterprise / org-specific needs
  • Requires app running locally
  • Subset of features
Approved clients only. Figma limits which MCP clients can connect. Claude Code, Cursor, Windsurf, Zed are on the approved list.
Section 2 — Setup

02

Setup in 3 steps

Get your Personal Access Token, add the config, restart your client.

↓ step by step
Setup · Step 1 — PAT

Step 1 — Get your Figma token

1
Open Figma → top-left menu → Settings
2
Go to Security → Personal access tokens → Generate new token
3
Name it (e.g. "claude-mcp"), set expiry, copy the token
Starts with: figd_...
Note: Token is shown once. Save it immediately — you can't retrieve it again.
Setup · Step 2 — Config

Step 2 — Add MCP config

Claude Code — settings.json

{
  "mcpServers": {
    "figma": {
      "command": "npx",
      "args": ["figma-mcp"],
      "env": {
        "FIGMA_ACCESS_TOKEN": "figd_your_token_here"
      }
    }
  }
}

Cursor / Windsurf — .cursor/mcp.json or .windsurf/mcp.json

Same JSON structure. File location differs per client — check their docs.
Setup · Step 3 — Verify

Step 3 — Verify connection

1
Restart your MCP client (Claude Code, Cursor, etc.)
2
Ask the agent: "Who am I in Figma?"
Agent calls: whoami()
3
Agent returns your Figma username and email
✓ Connection confirmed — all 19 tools are live
Tip: Install the Figma skills for your client before using write-to-canvas tools. Skills are required for use_figma and generate_figma_design.
Section 3 — Tools

03

19 Tools — Full Reference

Grouped into three categories: Read (9), Write (5), Code Connect (5).

↓ read → write → code connect
Tools · Read (9 tools)

Read tools — extract from Figma

Tool Returns
get_design_context Full node tree: layout, styles, components, constraints. Core tool for design-to-code.
get_variable_defs Figma Variables as structured tokens — colors, spacing, typography.
get_screenshot Renders a frame as PNG — visual reference for the agent.
get_metadata File name, pages, last modified, collaborators, version.
get_libraries Shared component libraries linked to the file.
search_design_system Search components and styles by name or description.
get_figjam Reads FigJam board content (sticky notes, connectors, shapes).
download_assets Exports icons and images from the file without manual export.
whoami Returns authenticated user name and email — use to verify setup.
Tools · Write (5 tools)

Write tools — create and modify in Figma beta free now

Tool What it does
use_figma Master canvas tool. Creates frames, components, auto layouts, text. Requires /figma-use skill first.
generate_figma_design Translates a page or layout description into native Figma frames. Higher-level than use_figma.
create_new_file Opens a new blank Figma file in your workspace.
generate_diagram Creates flowcharts and diagrams in FigJam from text or code.
upload_assets Pushes local images, icons, or SVGs into a Figma file.
Pricing note: Write-to-canvas is free during beta. Will become usage-based paid.
Tools · Code Connect (5 tools)

Code Connect — map design to codebase

Tool What it does
get_code_connect_map Shows current mappings: which Figma component → which code component.
get_code_connect_suggestions AI scans your codebase and suggests component mappings automatically.
get_context_for_code_connect Gets the Figma component context needed to create a mapping.
add_code_connect_map Adds a new mapping (Figma component → your actual code component).
send_code_connect_mappings Pushes all pending mappings to Figma — makes them available to all agents on the project.
Why this matters: Without Code Connect, the agent generates a generic <Button>. With it, it uses your <Button variant="primary"> from your component library.
Section 4 — Workflows

04

3 Core Workflows

Design → Code · Code → Design · Token Extraction

↓ workflow by workflow
Workflows · Design → Code

Workflow 1 — Design to Code

1
Copy the Figma frame/node URL or file key
Right-click frame → Copy link → paste into your prompt
2
Agent calls get_design_context + get_variable_defs
Reads all tokens, component tree, spacing, constraints
3
Agent generates component code
Exact hex values, exact spacing, exact font weights — not guesses
4
(Optional) Code Connect ensures your actual components are used
get_code_connect_suggestions → add_code_connect_map → send_code_connect_mappings
Workflows · Code → Design

Workflow 2 — Code to Design

Prerequisite: Run /figma-use skill before using use_figma.
1
Describe what you want, or point to existing code
"Create a login card with our design system tokens in Figma"
2
Agent reads get_variable_defs + get_libraries for context
Picks up your actual tokens before drawing anything
3
Agent calls use_figma or generate_figma_design
Creates native Figma frames, auto-layout, components
4
Frame appears in your open Figma file
Editable like anything you'd draw manually
Workflows · Token Extraction

Workflow 3 — Token Extraction

Pull design tokens from Figma Variables into your codebase — no manual copy-paste.

1
Agent calls get_variable_defs(fileKey)
Returns all Figma Variable collections (colors, spacing, typography...)
2
Agent transforms output to your target format
CSS custom properties · Tailwind config · DESIGN.md · tokens.json (DTCG)
3
Agent writes the file to your repo
Tokens are now in code — single source of truth from Figma
Combine with DESIGN.md: Extract tokens from Figma → populate DESIGN.md automatically. Figma stays the live source; DESIGN.md becomes the offline cache for agents.
Section 5 — Limits & Tips

05

Limits, Gotchas & Tips

What works, what doesn't, what to watch out for.

↓ details
Limits & Tips · Limits

Known limits

IssueDetail
Large files are slow Heavy frames with many nested components increase response time. Work with specific node IDs, not entire pages.
Approved clients only Only clients on Figma's MCP Catalog can connect. Building a custom client requires joining the waitlist.
use_figma needs a skill Must run /figma-use skill before calling use_figma. Skipping it causes tool errors.
Write is beta → paid Canvas writing is free during beta. Pricing model TBD — plan for usage-based billing.
Alpha spec Tool signatures and behavior may change. Pin your figma-mcp version in production.
Limits & Tips · Tips

Tips for best results

  • Pass node IDs, not full page URLs — narrows context, faster responses
  • Set up Code Connect before any design-to-code runs — ensures your real components are used
  • Run get_variable_defs first on any new file — gives the agent full token awareness before it generates anything
  • Use get_screenshot to give the agent a visual sanity check alongside get_design_context
  • Pair with DESIGN.md — Figma MCP for live reads, DESIGN.md as offline token cache
  • Keep your PAT scoped to read-only unless you're using write tools — reduces risk
Workflow shortcut: "Read my Figma file [URL] and generate a DESIGN.md from its variables" — chains get_variable_defs → DESIGN.md in one prompt.
