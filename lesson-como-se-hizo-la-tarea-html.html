La gran idea

Describís qué querés, la IA escribe el código, vos revisás en el browser y pedís ajustes. Eso es todo el proceso.

Metáfora: Es como pedirle a un carpintero muy hábil que haga un mueble. Vos describís el estilo, el color, si tiene cajones o no. Él construye. Vos aprobás o pedís cambios. No necesitás saber carpintería.

¿Cómo se hizo paso a paso?

1

Borrador en texto plano. Se escribió el contenido en Markdown — como un documento de Word pero más simple. Solo el texto, sin diseño.

2

Pedido de conversión. Se le dijo a Claude: "convertí esto a HTML con el mismo estilo oscuro de las otras lecciones." Claude vio los archivos existentes y copió el estilo.

3

Revisión y ajustes. Se abrió en el browser, se pidieron cambios: nuevo título, más ejemplos, ejemplos colapsables, campo de tono/estilo. Cada pedido en lenguaje natural.

4

Deploy a GitHub. Un comando y el archivo quedó publicado en internet con una URL para compartir.

Antes y después

❌ Antes — texto plano

Un archivo .md con el título, las preguntas y las secciones escritas en texto sin ningún formato. Legible, pero no presentable.

✅ Después — HTML diseñado

Mismo contenido. Dark theme, colores, checkboxes, 7 ejemplos desplegables, tabla de dominios, responsive en mobile, listo para imprimir.

Lo que se le pidió a la IA "Convertilo a HTML. Mismo estilo oscuro de las otras lecciones. Agregá más ejemplos, uno para ingeniero agrónomo con invernaderos, uno para app fitnessparkinson, uno para agente inmobiliario…"
